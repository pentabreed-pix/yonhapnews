Design System

Our design system allows for a unified design language
based on a foundation of consistent UI components.

Foundation

파운데이션은 일관된 레이아웃과 그에 따른 사용자 경험을 만드는 데 필수적인 시각적 요소입니다.
디자인의 기본이 되는 공통된 가치와 원칙을 가이드라인에 담아 진실의 중심으로 제공하고, 서비스가 통일된 디자인 언어에 맞도록 도움을 줍니다.

Components

컴포넌트는 디자인시스템의 재사용 가능한 구성요소입니다. 각 구성요소는 상호 작용 또는 UI 요구를 충족합니다.
컴포넌트 가이드라인은 컴포넌트의 이름을 통일하고 편리한 커뮤니케이션을 위해 구조, 용도, 유형, 패턴을 정의합니다.

Navigation

내비게이션(Navigation)은 사용자가 필요한 것을
찾도록 도와주며 현재 위치를 알려주는 컴포넌트입니다.
Button

버튼(Buttons)은 사용자의 성공적 작업 수행을 위해 사용되는 컴포넌트입니다. 시각적 계층 구조에 따른 차이가 있습니다.
Tag

태그(Tag)는 사용자에게 정보를 강조하거나 특정 요소에 주의를 끌기 위해 사용되는 컴포넌트입니다.
Input

인풋(Input)은 사용자가 UI상 텍스트 입력 및 확인 시 활용되는 컴포넌트입니다.
Table

테이블(Table)은 서비스내에서 사용되는 정보를 표 형식으로 보여주는 컴포넌트입니다.
List

리스트(List)는 텍스트 기반 정보를 섹션 또는 그룹으로 나눌 수 있는 연속적인 수직 집합체 입니다.
POPUP

팝업(POPUP)은 앱 콘텐츠 앞에 나타나 사용자에게 중요한 정보를 보내거나 사용자 선택을 요청하는 모달 유형의 컴포넌트입니다.
UI Elements

UI 엘리먼츠(UI Elements)는 UI 디자인에 필요한 구성요소입니다.

Guidelines

가이드라인은 서비스 고유의 일관된 UX를 유지하기 위해 만들어졌습니다.
서비스의 특성과 사용자 경험을 고려한 의미 있는 UX 가이드라인을 제시합니다.

Principles

모든 의사 결정은 사용자를 중심으로 이루어져야 합니다. 사용자 요구 사항을 기반으로 설계하고, 사용자가 원하는 정보와 기능을 제공하고 사용자가 목적을 달성하는 과정을 적극적으로 도와야 합니다.
Content design

콘텐츠 디자인을 할 때에는 UX 작성 및 정보 설계를 통해 UI를 사용자가 필요할 때 필요한 정보를 쉽게 사용할 수 있도록 디자인 합니다.
Principles

디자인 토큰은 색상, 타이포그래피, 간격 또는 고도와 같은 스타일을 할당하는 데 사용되는 저장된 값입니다. 토큰 시스템을 사용하면 공통 언어를 사용하고 플랫폼과 분야에서 일관된 디자인을 보장할 수 있습니다.
